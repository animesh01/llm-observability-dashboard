# Conversational AI Observability — Model-Health Dashboard

A runnable model-health dashboard for a customer-facing **agentic AI assistant**: one screen that answers *"is the assistant healthy right now?"* — organized around five buckets (**Quality · Safety · Performance · Cost · Drift**), with an executive summary and one-click export to PDF, PowerPoint, or clipboard.

The worked example tracks a production conversational-AI system over 30 days — including a synthetic incident (a hallucination spike on day 22), a new model release on day 18, and a slow intent-drift trend — so the dashboard tells a real operational story, not just static numbers.

## Data Disclaimer

All telemetry, metrics, domains, and alerts in this repository are **fabricated for demonstration**. The data is generated by `scripts/generate_telemetry.py`. Nothing here is derived from any employer, customer, or production system, and the repository contains no proprietary, confidential, or company-specific data.

## The five buckets

**Quality** — intent accuracy, resolution rate, first-contact resolution, repeat-contact rate, unanswered rate, turns-to-resolution, CSAT. (Resolution is tracked separately from containment, because a bot can avoid a human while still failing the customer.)

**Safety** — hallucination risk, groundedness / citation rate, policy-violation rate, PII flags, jailbreak / prompt-injection attempts blocked, refusal / over-refusal rate.

**Performance** — median latency, time-to-first-token, p95 latency (vs a 3-second SLA), tool-call failure rate, uptime, throughput.

**Cost** — cost per conversation, cost per *resolved* conversation, tokens per conversation, cache-hit rate.

**Drift** — intent drift, topic drift, quality drift, with a model-release marker so step-changes can be attributed to a specific deployment.

Plus an **Executive summary** (health verdict, what changed, top recommendation), **By domain** (the seven journeys scored side by side, plus crowd-vs-predicted intent agreement), **Failure analysis** (failed conversations bucketed by root cause), and **Alerts** (severity-ranked, bucket-tagged).

## Export & share

The executive summary tab has three working actions:

- **Copy executive summary** — renders the summary as plain text to copy into an email.
- **Download PDF report** — a one-page formatted PDF (executive summary + key numbers + per-bucket verdicts + recommendation).
- **Download PowerPoint deck** — a slide deck (title, summary, one slide per bucket, recommendation).

> Note on email: a public Streamlit app can't send mail without storing credentials, so the "email" action copies a ready-to-paste summary rather than sending directly. Wiring a real SMTP/SendGrid sender is a one-function swap if deployed internally.

## Run locally

```bash
pip install -r requirements.txt
streamlit run streamlit_app.py
```

To regenerate the synthetic dataset (optional):

```bash
python scripts/generate_telemetry.py
```

## Files

```
streamlit_app.py                 # the dashboard
scripts/generate_telemetry.py    # synthetic telemetry generator
scripts/report_builders.py       # PDF / PPTX / text export builders
data/telemetry.json              # generated demo dataset (30 days, 7 domains, 30 metrics/day)
```

## Portfolio scope

Intentionally lightweight: no backend, no production data dependencies. It exists to communicate conversational-AI observability, metric design across quality/safety/performance/cost/drift, and the product judgment to act on model-health signals — readable end to end by a product manager.
